Vanilla JS, CSS, and HTML

Wispr’s JS tab outputs the trail engine. The CSS tab styles particles; the HTML tab wraps both in one paste-ready block. Use that split or the combined snippet — whichever fits your build.

Workflow

  1. Open the playground and dial in presets, shapes, colors, chase, blend, and glow.
  2. Copy JS + CSS into your page (or only the HTML embed if you want a single drop-in).
  3. Load scripts late (e.g. before </body>) so layout is ready.
  4. Test motion on target devices; shorten the trail or reduce glow if frames dip.

Why Wispr for static sites

You see the exact trail before you ship. The same sliders control what lands in your export — no hand-tuning magic numbers in DevTools first.
